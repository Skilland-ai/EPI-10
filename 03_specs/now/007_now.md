## Canonical Boundary Rules
The v2 package must enforce these rules:

- Copilot is inside the AWS Espana / EPI10-owned boundary.
- Copilot is a harness/workflow, not a SaaS product boundary.
- Copilot consists conceptually of:
  - subagents;
  - skills;
  - workflow/scripts;
  - input checklist;
  - anonymization/pseudonymization step;
  - draft generation/support;
  - human review and approval gate;
  - export/handoff procedure.
- App Codigo is a possible interaction surface, not the architectural owner of
  the process.
- Runtime specifics are `Unknown` unless source evidence exists.
- Copilot still must not:
  - perform autonomous diagnosis;
  - make final approval decisions;
  - bypass human review;
  - store raw genetic data in PostgreSQL/Odoo;
  - log sensitive payloads or prompts without an approved policy.
- Healthie remains portal hypothesis/gate.
- Odoo remains operational backoffice/source of operational visibility.
- TellmeGen remains external/manual in Fase 1.
- PostgreSQL remains technical metadata only.

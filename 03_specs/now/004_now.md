## Interactive Phase Protocol

For each phase:

1. Read the phase artifacts.
2. Extract CTO-relevant facts, decisions, unknowns, risks, and assumptions.
3. Decide whether questions are needed before moving on.
4. Ask only questions that could change architecture, system design, scope,
   risk, sequencing, or solo-developer feasibility.
5. If a question is useful but not blocking, record it as `Unknown` and proceed.
6. If a question is blocking, stop and ask the user before continuing.
7. Record questions and answers in scratch notes or directly in the final
   `Questions and answers log`.

Use concise question batches. Prefer at most 5 questions per phase unless there
is a hard blocker.

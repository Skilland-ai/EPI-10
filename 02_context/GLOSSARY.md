# GLOSSARY

- Term: Fase 0
  Definition: Fase 0 is the internal pre-sales/research phase used to understand feasibility and prepare a serious proposal; it is not sold as a standalone client phase.
- Term: Fase 1
  Definition: Future client-facing implantation phase for the MVP operativo once the research and proposal are ready.
- Term: MVP operativo
  Definition: The first practical operating system for EPI10, focused on running the service with existing tools and minimal custom development.
- Term: Gemelo digital
  Definition: Longer-term product vision based on genetic, habit, analytics and personalized follow-up data; explicitly not the current project scope.
- Term: Odoo
  Definition: Open source management platform being evaluated as possible CRM/backoffice/core system for EPI10 operations.
- Term: TellmeGen
  Definition: Genetic-results provider whose real API availability, data scope and integration constraints must be validated.
- Term: Healthie
  Definition: Example third-party platform to evaluate for portal, forms, communication and client-operational workflow coverage.
- Term: Continuous Care
  Definition: Second third-party platform to evaluate for patient/client workflow coverage and secure operational features.
- Term: GDPR
  Definition: European data-protection framework used in this repo as a gating requirement for any tool handling sensitive health or genetic data.
- Term: Build vs buy
  Definition: Decision frame comparing custom development against configuring or integrating existing tools.
- Term: Semiautomático
  Definition: Operating model where part of the flow is automated but explicit human intervention remains acceptable in the MVP.
